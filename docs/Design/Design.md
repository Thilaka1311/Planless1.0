# Planless Design System

Version: 1.0

This document defines the overall visual language of Planless.

Unlike colors.md and typography.md, this document does not define tokens.

Instead, it explains how the application should use typography, colors, spacing, motion, imagery, hierarchy, and interaction to create one cohesive experience.

This is the primary design document every screen should follow.

colors.md defines color.

typography.md defines type.

design.md defines how everything comes together.

---

# Philosophy

Planless is not a productivity application.

It is a social product.

The interface should encourage people to spend less time managing plans and more time enjoying them.

The application should feel calm while planning.

Exciting while participating.

Beautiful without trying to impress.

The UI should disappear behind the experience.

People should remember the football match.

Not the button they tapped.

---

# Design Principles

Everything should follow five principles.

## 1. Content First

The plan is always the hero.

Never decorate around empty space.

Remove unnecessary UI before adding more.

---

## 2. Hierarchy Before Color

Users should understand importance through

• spacing

• typography

• size

before color.

Color reinforces hierarchy.

It never creates hierarchy.

---

## 3. One Hero Per Screen

Every screen has one dominant element.

Examples

Large event image

Match score

Movie poster

Restaurant photo

Countdown

Never compete with the hero.

Everything else supports it.

---

## 4. Motion Creates Delight

Animation should explain.

Never decorate.

Motion should communicate

expansion

selection

navigation

confirmation

completion

Never animate simply because something changed.

---

## 5. Invisible Interface

The interface should never feel designed.

It should simply feel obvious.

The fewer UI elements needed to accomplish something,

the better.

---

# Platform Identity

Planless is monochrome.

Black.

White.

Gray.

The platform should never have a color identity.

Experiences provide color.

The platform provides consistency.

A user should always recognize Planless,

even if every category disappeared.

---

# Experience Identity

Every experience receives

its own

accent

imagery

personality

motion

without changing the platform.

Examples

Sports

Movies

Restaurants

Future categories inherit the same platform.

Only the experience changes.

---

# Layout

Whitespace creates hierarchy.

Not borders.

Not shadows.

Every layout should follow

Hero

↓

Primary Action

↓

Important Information

↓

Supporting Information

↓

Metadata

Never reverse this order.

---

# Spacing

Spacing is one of the strongest branding tools.

Prefer generous spacing.

Avoid crowded layouts.

Every screen should breathe.

Increase spacing before increasing borders.

Increase spacing before increasing shadows.

---

# Cards

Cards are containers.

Not decorations.

Cards exist only to group related content.

Every card should contain one purpose.

Examples

Plan Card

Restaurant Card

Movie Card

Participant Card

Settings Group

Avoid nested cards whenever possible.

---

# Surfaces

Use elevation instead of borders.

Base Surface

↓

Elevated Surface

↓

Floating Surface

↓

Overlay

Never stack many elevated surfaces.

Depth should remain obvious.

---

# Borders

Borders are optional.

Whitespace is preferred.

Only introduce borders when separation becomes unclear.

Borders should remain visually quiet.

---

# Photography

Photography is the emotional layer of Planless.

Whenever imagery exists,

prioritize imagery over illustration.

Sports

show people.

Movies

show posters.

Restaurants

show food.

Photography should communicate emotion.

The interface should frame it.

---

# Icons

Icons communicate actions.

Typography communicates information.

Never use icons to replace readable labels.

Icons should remain simple.

Avoid decorative iconography.

---

# Buttons

Buttons communicate priority.

Not brand.

Every screen should contain

one primary button.

Secondary actions should remain visually quieter.

Avoid placing multiple equally strong CTAs together.

---

# Inputs

Inputs should feel native.

Never over-style form controls.

Focus should feel immediate.

Typing should never feel visually heavy.

---

# Lists

Lists are the foundation of Planless.

Lists should feel effortless to scan.

Every row should have

Primary information

↓

Secondary information

↓

Metadata

Maintain consistent alignment throughout.

---

# Motion

Motion should feel physical.

Everything has weight.

Sheets slide.

Cards expand.

Buttons compress.

Lists reorder smoothly.

Never teleport elements.

Always preserve spatial continuity.

---

# Gestures

Support natural mobile interactions.

Tap

Press

Drag

Swipe

Long Press

Never require hidden gestures.

Every gesture should have a visible alternative.

---

# Empty States

Every empty state should explain

what happened

why

what to do next

Avoid generic illustrations.

Avoid generic copy.

Encourage action.

---

# Feedback

Every user action deserves feedback.

Buttons

compress.

Selections

highlight.

Completed actions

celebrate subtly.

Errors

explain the solution.

Never leave the user wondering whether something happened.

---

# Loading

Loading should preserve layout.

Avoid layout jumps.

Prefer skeletons.

Avoid infinite spinners whenever possible.

---

# Accessibility

Readable typography.

Comfortable touch targets.

High contrast.

Consistent spacing.

Every interaction should remain usable with one hand.

Animations should never block interaction.

---

# Design Quality Checklist

Before shipping any screen ask:

Is there one hero?

Can anything be removed?

Does spacing create hierarchy?

Are colors semantic?

Is typography consistent?

Can this screen exist without decoration?

Does the experience feel social?

Would Apple remove anything?

Would the interface still feel premium in grayscale?

If the answer is yes,

the design is likely correct.

---

# Golden Rule

Planless should never feel designed by AI.

It should feel designed by restraint.

Whenever unsure,

remove instead of add.

Simplify instead of decorate.

Content should always be more memorable than the interface.
